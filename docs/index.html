How it works

Three moves, and it fits how you already work

Most repos already have a plan — plan.md, tasks.md, an agent's todo list. Trackly reads it, tracks execution against it, and shows how far along you are.

1

Capture the plan

The agent pushes its plan as a markdown checklist or JSON. Or Trackly seeds one from a doc already in your repo.

2

Track execution

As tasks land, they're marked done, partial, or blocked — with an optional commit hash or note as evidence.

3

Report it

trackly status for a terminal scoreboard; trackly report for a clean HTML doc you print to PDF.

Installation

Install once, use in every repo

Trackly is a single native binary. End users never need Rust. One line on macOS / Linux — or grab a prebuilt archive for any platform.

install — macOS / Linux
# downloads the latest prebuilt binary for your platform
curl -fsSL https://raw.githubusercontent.com/vehutech/trackly/main/scripts/install.sh | sh

# verify (installs `trackly` and the `trackly-mcp` agent server)
trackly --version
trackly 0.3.0

Prefer to do it by hand? Download an archive from the latest release — aarch64/x86_64 macOS, x86_64 Linux, and x86_64 Windows — or build from source:

build from source
git clone https://github.com/vehutech/trackly
cd trackly
cargo build --release -p trackly-cli
cp target/release/trackly /usr/local/bin/   # or anywhere on PATH

CLI reference · the API

Every command

An agent that can run a shell can drive all of this. So can you.

CommandWhat it does
trackly initCreate .trackly/, seeding a plan from an existing doc if one is found.
trackly plan set <file|->Replace the plan from a markdown checklist or JSON (- = stdin).
trackly plan add "<title>"Append one task. Flags: --group, --weight.
trackly task done <id>Mark a task done. Also: partial, start, block, open. Flag: --evidence.
trackly statusPrint the terminal scoreboard — %, done/partial/open, grouped.
trackly reportWrite a print-ready HTML report. Flags: --out, --subtitle.
Plan format & scoring

Ordinary markdown. Nothing new to learn.

Plan format

# Payments Service

## Phase 1 — Foundations
- [x] Scaffold the service   ← done
- [/] Set up CI              ← in progress
- [ ] Write the docs         ← open

## Phase 2 — Core flows
- [~] Webhook receiver       ← partial
- [-] Fraud check            ← blocked
[x] done [ ] open [/] in progress [~] partial [-] blocked

Scoring — line-weighted, half-credit partials

percent = Σ(credit × weight) / Σ(weight) × 100

credit:  done = 1.0
         partial = 0.5
         open / in‑progress / blocked = 0

Every task defaults to weight 1; bump --weight for larger items. The same method noted in the reference report: "overall (half-credit partials)." Treat the % as a rough directional signal, not a precise measure of effort.

Repository layout

crates/
  trackly-core/   # model, store, scoring,
                  # parsing, report — UI-agnostic
  trackly-cli/    # the `trackly` command
src-tauri/, src/  # future desktop app

What's stored — .trackly/

.trackly/plan.json      # the plan (source of truth)
.trackly/history.jsonl  # append-only snapshots
                        # (the trend over time)

Both are human-readable. Commit them to version the plan and its history.

Roadmap

Where it's going

NOW
v0.1 — the CLI
Plan capture, task tracking, terminal status, HTML/PDF report.
NOW
MCP server
Ships as trackly-mcp — agents (Claude Code, Cursor, Windsurf) call set_plan / update_task natively. claude mcp add trackly -- trackly-mcp
NOW
Prebuilt binaries
Cross-platform release archives for macOS, Linux & Windows + a one-line installer — zero Rust. brew/npx next.
NOW
Git as evidence
Opt-in post-commit hook: commits that mention a task id auto-record the commit as evidence and advance the task. trackly hook install
NOW
The "like GitHub" desktop app
A native app (Tauri + React) that discovers every tracked repo on your machine, shows a dashboard, and exports reports. npm run tauri dev — prebuilt installers next.
Contributing

cargo build

cargo clippy --all-targets
